Week 09 Notes

The Window Object

Every browser window, tab, popu, frame, and iframe has a window object.

In a browser environment, the browser object and the global object are the same thing.

Find the global object by using 'this' in a global scope.

Going Global

Global variables are properties of a global object.

Functions like parseInt() are methods of global object.

Except some of these are part of the number object now in es6.

window.alert, confirm, and prompt

Don't use these too much. Maybe to confirm if a user wants to delete a resource.

Which Browser?

accessing the navigator window property will return Navigator object, containing info about the browser.

This is not reliable though.

Location

window.location returns an object with infor about the current page url.

href property retursn url. It can be rewritten to direct the page to a new url.

reload() method can force a page reload. Given parameter of true, it will reload from the server instead of using cached page.

assign() will load another resource from url. replace() will do the same thing, but won't store it in history.

Browser History

window.history lets you access history from previously visited pages in the current browser session. different from html5 history api.

window.history.go() is used to go to a specific page, 0 is the current page

Controlling Windows

Use window.open(url, title, attributes) lets you open a new window.

window.moveTo(), move the window to the top-left corner of screen.

Popups are very annoying

Screen Information

find info about the screen width and height

The Document Object

Every window contains a document

document.write() puts some text on the page, replacing the whole document if there is stuff there

You can use document.write() in a script tag to inject a string into markup

Cookies

small files saved locally on a user's computer

Used to remember things about previous visits to a page

cookies are often replaced by localstorage

Still used to pass info between client and server on every https request

Cookie example: "name=Superman; hero=true; city=Metropolis"

In EU you must ask permission before setting cookies

Creating Cookies

put the cookie in the cookie jar.

Cookie expiry dates

cookies are session cookies by default.

But you can add an expiration date to make them last longer

The path and domain of cookies

By default, cookies can only be read by pages inside the same directory and domain as the file was set. But you can change the path so that any page in the root directory can read the cookie.

Secure cookies

Add 'secure to the end of a cookie to make sure it's only transmitted over https

Deleting Cookies

You must set it to expire in the past to remove a cookie.

Timing Functions

setTimeout takes callback as first parm, num of milliseconds as second parm

setInterval repeatedly calls the callback every given number of milliseconds

Careful about using 'this' in timing methods. Binding is to the window, not the method's object

Animation

You can use the timing function to animate elements on page

requestAnimationFrame

Much smoother animation performance than using timeout for animation

HTML5 APIs

The data Attribute

Used for embedding data using custom attributes that are ignored by the browser

They are used by javascript, not the browser

Each element has a dataset property that can be used to acces data attributes

Superman

HTML5 api's

Use feature detection before using api's

Web storages

different from cookies because:

  • information is not shared with the server on every request
  • Information is available on multiple windows of the browser for the same domain
  • Storage capacity limit is larger than cookies
  • Data does not automatically expire.

Local storage and session storage

Setting something to local storage fires an event.

Geolocation

Obtain geographic position of device

Use to link to nearby places or measure the speed they are moving

navigator object has a method called getCurrentPosition() that returns position object to a callback function

Web Workers

allows processes to be run in the background

don't let long scripts make your page unresponsive

Websockets

Allows two way server communication: push messaging

When websocket successfully connects to a url, it fires an event called 'open'

Notifications

You need permission to use notifications.

notifications have events to react to, like click, and close.

You can open a new window when they click on it.

Multimedia

audio and video html tags

Methods that go with them to set controls, etc.

Other api's.

There are so many more useful things out there.

Shims and Polyfills

shims and polyfills fill in the code for you when it's not working natively on the browser.
