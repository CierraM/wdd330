JS Ninja Chapter 10: Testing and Debugging

Errors, Exceptions, and Warnings

Three types of errors:

  • System error: problem with system or external devices
  • Programmer error: bad syntax or logic
  • User error: user entered wrong data, program can't handle it.

User errors might also be programmer errors, because you should design the program in a way that will prevent a user error.

Exceptions

an exception is an error that produces return value to help you deal with it.

Stack Traces

stack trace: the trail of functions or methods that lead to the point where the error occured.

Warnings

Small errors that don't crash the program.

The Importance of Testing and Debugging

Make sure your code fails loudly so you can identify and fix the problem.

In production, let it fail gracefully so as to minimally impact user experience.

Strict Mode

Strict mode helps you find errors and problems faster.

Linting Tools

Helps you not be a sloppy programmer.

Good for working with team, since it makes sure everyone writes code the same way.

Error Objects

You can use constructor function to make your own errors

The parameter for error constructor is the error message.

Or you can use one of the 7 specific error objects:

  • EvalError: used to find errors in the global eval() function
  • RangeError: when a number is outside allowable values
  • ReferenceError: reference is made to something that doesn't exist
  • SyntaxError: syntax problem.
  • TypeError: problem with type of value that is being used
  • URIError: problem encoding/decoding url
  • InternalError: problem in javascript engine (too much recursion)

You can use constructor function to make any of these errors happen.

Throwing Exceptions

Use throw keyword to stop the program

You can throw anything, but it is best to throw an error object.

Exception Handling

handle exceptions gracefully by catching an error rather than letting the program crash (this is for production)

try, catch, and finally

If you suspect it will result in exception, wrap it in a try block. It will then be passed to the catch block.

finally block can go after catch block. Catch is only executed if try fails, but finally is always executed at the end.
