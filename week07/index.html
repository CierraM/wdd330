Chapter 11: Further Functions

Further Functions

Functions are first-class objects.

They have properties and methods.

Call and Apply Methods

call(): sets the value of 'this' inside a function to an object that is provided as the first argument

If you don't need to change 'this' you can pass null as the first argument.

Apply works the same way, but the arguments that go to that function are provided as an array, where in call(), they are just passed after the first argument.

These are useful because they let you write generalized functions not tied to specific objects.

Custom Properties

You can add properties to functions just like you can to objects.

Example: square.description = 'description'. Add description property to a function that describes what it does.

Memoization

If a function takes a bit to compute a value, save the result in a cache property.

Same thing? https://developer.mozilla.org/en-US/docs/Web/API/Cache

Immediately Invoked Function Expressions (IIFE)

Anonymous function that's called as soon as defined.

put parentheses at the end, and put the whole function inside parentheses so it counts as an expression.

Useful for performing a task, while keeping variables wrapped up in function scope.

Temporary Variables

Place a variable that you want to be temporary inside an IIFE

Initialization Code

Use an IIFE to set up your code variables, objects, event listeners, etc.

But this isn't actually that useful in es6 because we can just use let and give things block scope.

Safe Use of Strict Mode

Use IIFE's to only apply strict mode to stuff you are coding.

Functions that Define and Rewrite Themselves

Assign an anonymous function to a variable with the same name as function

The catch: If the function is assigned to a variable, that variable will hold onto the first function.

This is useful if you want some different code the first time it is invoked to initialize things.

Init Time Branching

Used with feature detection to create functions that rewrite themselves to do what they can do depending on the browser.

Recursive Functions

Recursive functions invoke themselves until a certain condition is met.

Functions that Return Functions

Closures

A reference to a variable that was created inside the scope of another function but is then kept alive and used in another part of the program.

When you make a function that returns a function, the new function will still have access to variables it needs from its outer environment.

Generators

ES6 Introduced generators.

Special functions used to produce iterators that maintain the state of a value.

Define with * after functino declaration.

When you call a generator, it doesn't run the code in the function, but it returns a generator object that can be used to create an iterator that implements a next() method that returns a value every time the next() method is caled.

yield keyword: used to return value for generator function.

the state of the yield value is remembered next time it is called.

Functional Programming

Pure Functions

  1. The return value of a pure function depends only on values given as args.
  2. There are not sideeffects that change data elsewhere in the program from one function. Only returns new data, doesn't alter old data.
  3. Refrential transparency: given the same arguments, it will return the same things.

All functions must have arguments and return value.

Using const to declare variables will help you avoid destructive data transformations.

Higher Order Functions

Accept as argument or return another function.

Curryings

When not all arguments are supplied to the function, so it returns another function that retains arguments already provided
